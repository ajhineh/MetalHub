# ADR-004: Phase 9 Tech Stack and Migration Strategy

**تاریخ**: ۲۰۲۶-۰۷-۲۰  
**وضعیت**: Accepted  
**تصمیم‌گیرندگان**: تیم تحقیق، یوسف رضایی، تیم معماری

## زمینه (Context)
با تکمیل پایگاه دانش معماری (Architecture Knowledge Base) مبتنی بر مدل C4 در فازهای قبلی، اکنون نیاز است تا زیرساخت و معماری نرم‌افزاری پروژه در فاز ۹ (Software Implementation) به سمت معماری مایکروسرویس و رویداد-محور (Event-Driven) حرکت کند. مخزن فعلی بر پایه Strapi CMS و Next.js بنا شده بود که برای پردازش‌های سنگین هوش مصنوعی، گراف دانش و مدیریت پیچیده تراکنش‌های صنعتی کافی نیست.

## تصمیم (Decision)
- **Core Service API**: استفاده از **NestJS** (با TypeScript) به دلیل پشتیبانی عالی از DDD (طراحی دامنه‌محور)، CQRS و ماژولار بودن.
- **Decision Engine**: استفاده از **Python + FastAPI** برای بهره‌برداری از اکوسیستم قدرتمند هوش مصنوعی (LangChain, LlamaIndex).
- **Message Bus**: استفاده از **RabbitMQ** برای شروع (سبک، با مدیریت Dead-Letter Queue) و مهاجرت به Kafka در آینده در صورت رسیدن به مقیاس بالا (بیش از ۱۰ هزار RFQ در روز).
- **API Gateway**: استفاده اولیه از Next.js/Vercel Edge Routes و مهاجرت به Kong/Nginx در مقیاس‌های بعدی.
- **استراتژی مهاجرت (Migration)**: استفاده از روش **Strangler Fig**. یعنی کدهای قبلی (Strapi) بلافاصله پاک نمی‌شوند، بلکه ابتدا Core API جدید به صورت موازی ساخته می‌شود و مسیرها به صورت تدریجی منتقل خواهند شد تا در صورت بروز مشکل امکان Rollback وجود داشته باشد.

## گزینه‌های بررسی‌شده (Alternatives Considered)
1. **استفاده از Express.js برای Core API**: بسیار سبک است اما برای یک سیستم Enterprise و اجرای الگوهای DDD و CQRS نیازمند توسعه زیرساخت‌های فراوان دستی بود (رد شد).
2. **استفاده از Apache Kafka از روز اول**: سربار مدیریت بسیار بالایی دارد و در فازهای اولیه MVP نیاز مبرمی به throughput بسیار بالای آن نیست (رد شد، RabbitMQ انتخاب شد).
3. **مهاجرت ناگهانی (Big Bang Rewrite)**: ریسک بسیار بالایی برای پایداری سیستم دارد (رد شد).

## عواقب (Consequences)
### مثبت
- قابلیت مقیاس‌پذیری و پشتیبانی از هوش مصنوعی بسیار بالا می‌رود.
- تفکیک وظایف (CQRS) باعث می‌شود کوئری‌های رابط کاربری سریع و مجزا از نوشتن داده‌های سنگین باشند.
- قابلیت Traceability با Event Sourcing در صنعت فولاد به شدت افزایش می‌یابد.

### منفی / ریسک‌ها
- پیچیدگی پیاده‌سازی و نیاز به مدیریت چندین Container به صورت همزمان (Docker Compose پیشرفته).
- زمان یادگیری بالاتر برای توسعه‌دهندگان جدید در خصوص NestJS و RabbitMQ.
