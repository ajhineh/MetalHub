# Experience Capture

## Goal
To automatically harvest insights, lessons learned, and performance data from ongoing projects to enrich the Industrial Memory. This capability ensures that every project execution continuously builds the platform's collective intelligence and trust ecosystem.

## Inputs
- Execution logs and system interactions
- QC reports, non-conformance reports (NCRs), and inspection data
- Milestone completion records and time-tracking data
- User communications and feedback within the platform

## Outputs
- Formatted lessons learned and best practices
- Dynamically updated supplier Trust Scores
- Reusable knowledge assets for future guidance
- Updates to the overarching capability graph

## Actors
- All project actors (Engineers, Suppliers, Logistics)
- System (AI Copilot / Knowledge Engine)

## Rules
- Experience must be captured in the natural flow of work without adding administrative burden to the users.
- Captured data must be objective, tying outcomes to specific decisions and evidence.
- Trust score updates must be driven by verified execution data, not subjective ratings alone.

## Success Metrics
- Volume and quality of captured insights per project
- Knowledge reuse rate in subsequent projects
- Accuracy of the Trust Graph updates
- User engagement with captured lessons learned
