# Tender Evaluation

## Goal
To objectively assess received proposals against validated requirements, identifying risks, inconsistencies, and true value to enable confident, defendable decision-making. This capability elevates the evaluation process from basic price comparison to comprehensive capability and risk assessment.

## Inputs
- Received supplier quotations (RFQs)
- Validated engineering requirements and drawings
- Supplier profiles and updated Trust Scores
- Historical pricing and lead-time data
- Commercial and technical constraints

## Outputs
- Consolidated evaluation and comparison matrix
- Risk flags and identified inconsistencies
- Value-based recommendations
- Decision logs and evidentiary trails

## Actors
- Procurement Manager
- Engineering Reviewer
- Client
- AI Copilot (Decision Context Engine)

## Rules
- Evaluations must include evidence-based scoring and explicitly highlight capability or compliance gaps.
- The system must require specific evidence (e.g., certifications, past QC reports) before allowing an evaluator to approve a tender.
- Price alone cannot be the sole deciding factor; risk and trust must be quantified.

## Success Metrics
- Reduction in tender evaluation time
- User decision confidence score
- Error detection rate in received proposals
- Long-term alignment between selected tenders and actual project outcomes
