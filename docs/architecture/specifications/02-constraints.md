# 02-Constraints

## Architectural Constraints
- **CON-001**: Avoid "Feature Creep" - Every feature must map to a reduction in uncertainty or an increase in organizational wisdom. [Source: Architecture Consolidation]
- **CON-002**: Avoid "Knowledge Debt" - Knowledge must not be lost or untraceable. [Source: Architecture Consolidation]
- **CON-003**: No Unverified Claims - The system is built on an event-driven "Commitment" model where every stage is a verifiable event (evidence-based). [Source: Architecture Consolidation]
- **CON-004**: Decision Traceability - Every commit/action must be linked back to a Mission/ADR/Capability. [Source: Architecture Consolidation]
- **CON-005**: Transparency Over Control - Governance is distributed and evidence-based. [Source: Architecture Consolidation]
