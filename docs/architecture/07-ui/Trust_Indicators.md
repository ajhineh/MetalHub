# Trust Indicators

## Philosophy
"Transparency Builds Trust." Users in the industrial sector operate in high-stakes environments. They need to know *why* the system recommends something and *where* the data comes from. Trust is earned through visible evidence.

## Core Concepts
- **Verifiable Provenance**: Every metric, score, or recommendation must visually link back to its source data.
- **Confidence Levels**: The system should visually express its certainty. If data is sparse, the UI should reflect lower confidence rather than feigning certainty.
- **Visual Distinctness**: Clearly differentiate between user-provided claims (unverified) and platform-validated evidence (verified).

## Interaction Logic
- **Evidence Badges**: Use distinct icons (e.g., a shield or checkmark) alongside verified data points. Clicking the badge opens a modal showing the evidence trail.
- **The "Why" Pattern**: Every recommendation from the `Guidance Engine` includes a "Why am I seeing this?" link, which explains the logic and data behind the suggestion.
- **Risk Visualization**: Use color (sparingly and intentionally) and charts to visualize the risk score generated by the `Decision Engine`. Ensure the breakdown of the score is easily accessible.
- **Audit Trails**: Provide a clear, chronological view of how a document or decision evolved over time, reinforcing "Decision Continuity".
